指定された範囲のバックログタスクに対して、start → apply → complete のサイクルを連続で回す。

**Input**: タスク範囲を指定する。例: `1-5`, `3-7`, `2-` (2番目以降全部), `-4` (最初から4番目まで)
引数がない場合は、未完了タスク全てを対象とする。

**Steps**

1. **バックログを読んでタスク一覧を取得**
   `docs/06-tasks.md` を読み、未着手セクション（`- [ ]`）のタスクを順番にリストアップする。
   各タスクにはインデックス番号（1始まり、未着手タスクの上から順）を振る。

2. **対象範囲を決定**
   ユーザーの引数を解析して対象タスクを決定する:
   - `1-5`: 1番目から5番目まで
   - `3-7`: 3番目から7番目まで
   - `2-`: 2番目から最後まで
   - `-4`: 1番目から4番目まで
   - 引数なし: 全ての未着手タスク

   対象タスクの一覧を表示し、処理を開始する旨を宣言する。

3. **タスクごとにサイクルを実行（ループ）**

   対象範囲の各タスクについて、以下を順番に実行する:

   **3a. Start**
   Skill toolを使って `osdf:start` を実行する。

   **3b. Apply**
   Skill toolを使って `osdf:apply` を実行する。

   **3c. Complete**
   Skill toolを使って `osdf:complete` を実行する。

   **3d. 進捗報告**
   ```
   ✓ タスク N/M 完了: <タスク概要>
   ```
   を表示し、次のタスクへ進む。

   **エラー時の挙動:**
   - いずれかのステップでエラーやブロッカーが発生した場合、そのタスクで停止する
   - 現在の進捗を報告し、ユーザーに判断を仰ぐ（続行 or 中断）

4. **完了報告**

   全タスクが完了したら、サマリーを表示する:
   ```
   ## Batch Complete

   **対象:** タスク N 〜 M
   **結果:** X/Y タスク完了

   ### 完了したタスク
   - [x] タスク1の概要
   - [x] タスク2の概要
   ...
   ```

**Guardrails**
- 各タスクは必ず start → apply → complete の順で実行する
- 1つのタスクが完全に完了してから次のタスクに進む
- エラー時は即座に停止してユーザーに報告する
- 各サイクルの間でバックログの状態が正しく更新されていることを前提とする
